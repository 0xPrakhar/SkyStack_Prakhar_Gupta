# Eventify

Eventify is a React + Vite event discovery and booking app with a Node.js + Express API. It supports JWT authentication, protected bookings, admin-only management, event search, filters, ratings, and optional public weather context from Open-Meteo.

## Stack

- Frontend: React, Vite, Tailwind CSS
- Backend: Node.js, Express
- Auth: JWT, bcryptjs
- Validation: Zod
- Public API: Open-Meteo weather data
- Deployment: Render blueprint

## Local Setup

### Backend

```powershell
cd server
Copy-Item .env.example .env
npm install
npm start
```

Backend URL: `http://127.0.0.1:5000`

Health check: `http://127.0.0.1:5000/api/health`

### Frontend

```powershell
npm install
npm run dev
```

Frontend URL: `http://127.0.0.1:5173`

## Default Admin Login

- Email: `admin@eventify.local`
- Password: `Admin@12345`

For local development only, the backend seeds this admin account if it does not already exist. In Render, `DEFAULT_ADMIN_PASSWORD` is generated by the blueprint, so replace these values in `server/.env` if you want a custom local admin.

## Security Notes

- `/event/:id`, `/library`, and `/my-events` require a logged-in user.
- `/admin` requires a logged-in admin user.
- Booking creation is enforced by the backend at `/api/bookings`.
- Admin overview and booking data are enforced by the backend at `/api/admin`.
- The Google/open-auth placeholder has been removed; authentication is handled by the app's own email/password API.

## Render Deployment

This repo includes `render.yaml`.

1. Push the repo to GitHub.
2. In Render, choose `New +` then `Blueprint`.
3. Connect the repo.
4. Render will create:
   - `eventify-db` for persistent PostgreSQL storage.
   - `eventify-api` for the backend.
   - `eventify-web` for the frontend.
5. After the first deploy, review the generated `DEFAULT_ADMIN_PASSWORD` value in Render and store it somewhere safe.

The blueprint wires:

- `DATABASE_URL` from the Render Postgres instance into the API
- `VITE_API_URL` from the API service URL into the frontend
- generated secrets for `JWT_SECRET` and the default admin password
